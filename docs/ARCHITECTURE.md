## Data Flow

### Read (Query) flow

```
Component mounts
       │
       ▼
useGetStudents()          ← generated by orval from OpenAPI
       │
       ▼
TanStack Query checks cache
       │
   ┌───┴────────────────┐
   │ STALE or MISSING   │ FRESH
   ▼                    ▼
Axios GET /api/students  Return cached data immediately
       │                 (background refetch if stale)
       ▼
Response → Query cache
       │
       ▼
Component re-renders with data
```

### Write (Mutation) flow

```
User submits form
       │
       ▼
React Hook Form validates via Zod
       │
   ┌───┴──────────┐
   │ INVALID      │ VALID
   ▼              ▼
Show errors    mapFormToDto()  ← student.mapper.ts
                   │
                   ▼
             useCreateStudent() mutation
                   │
                   ▼
             Optimistic update → cache
             Axios POST /api/students
                   │
              ┌────┴──────────┐
              │ SUCCESS       │ ERROR
              ▼               ▼
        Invalidate cache   Roll back optimistic
        Show success toast mapApiErrorsToForm()
        Navigate to list   Show error toast
```

## Auth & Token Refresh Flow

```
App loads
    │
    ▼
AuthProvider.useEffect()
    │
    ├── accessToken in store?
    │       │
    │       YES → syncMe()
    │              │
    │              ├── GET /api/auth/me
    │              │       │
    │              │    SUCCESS → setUser(data)
    │              │    ERROR   → clearAuth() → redirect /login
    │              │
    │       NO  → stay on /login
    │
    ▼
User logs in
    │
    ▼
POST /api/auth/login
    │
    ▼
setTokens({ accessToken, refreshToken })
setUser(me)
    │
    ▼
Redirect to /<role>/dashboard

─────────────────────────────────────────────────

Any request returns 401
    │
    ▼
Axios response interceptor fires
    │
    ├── isRefreshing?
    │       │
    │       YES → push original request to failedQueue, wait
    │       NO  → set isRefreshing = true
    │              │
    │              ▼
    │           POST /api/auth/refresh  (with refreshToken)
    │              │
    │         ┌────┴────────────────┐
    │         │ SUCCESS             │ FAILURE
    │         ▼                     ▼
    │    setTokens(newTokens)   clearAuth()
    │    isRefreshing = false   redirect /login
    │    flush failedQueue      reject all queued
    │    (retry each with       requests
    │     new token)
    │
    ▼
Original request retried → response returned to component
```

## WebSocket Event Pipeline

```
Login success
    │
    ▼
socket.connect(wsUrl, { auth: { token: accessToken } })
    │
    ▼
socket.on('connect')
    │
    ▼
Register event handlers:

  'student:created'  → queryClient.invalidateQueries(['students'])
  'student:updated'  → queryClient.setQueryData(['students', id], dto)
  'student:deleted'  → queryClient.invalidateQueries(['students'])

  'payment:received' → queryClient.setQueryData(['invoices', id], dto)
                     → addToast({ type: 'success', message: ... })

  'homework:graded'  → queryClient.setQueryData(['homework', id], dto)
                     → addToast({ type: 'info', message: ... })

  'attendance:saved' → queryClient.invalidateQueries(['attendance'])

─────────────────────────────────────────────────

Logout / clearAuth()
    │
    ▼
socket.disconnect()
Remove all event listeners
```

## Multi-Tenancy Flow

```
Browser navigates to bright.educrm.uz
    │
    ▼
Next.js middleware reads hostname
    │
    ▼
TenantProvider mounts
    │
    ▼
GET /api/tenant/config?subdomain=bright
    │
    ▼
Response: { config: {...}, featureFlags: {...} }
    │
    ▼
useTenantStore.loadTenant(payload)
    │
    ├── Store config (name, logo, primaryColor, currency, ...)
    ├── Store featureFlags
    └── Inject CSS custom properties:
            document.documentElement.style.setProperty(
              '--color-primary', config.primaryColor
            )
            ...etc
    │
    ▼
All components now use:
  var(--color-primary)      → school brand color
  var(--color-secondary)    → accent color
  useTenantStore().config   → school name, logo
  useTenantStore().featureFlags.homeworkModule  → show/hide homework
```

## RBAC Decision Tree

```
can(user, permission)
    │
    ├── user is null/undefined?
    │       └── return false
    │
    ├── user.role === 'owner'?
    │       └── return true  (owner can do everything)
    │
    ├── user.role === 'admin'?
    │       ├── permission in ADMIN_PERMISSIONS?
    │       │       └── return true
    │       └── else return false
    │
    ├── user.role === 'teacher'?
    │       ├── permission in TEACHER_PERMISSIONS?
    │       │       └── return true
    │       └── else return false
    │
    └── user.role === 'student'?
            ├── permission in STUDENT_PERMISSIONS?
            │       └── return true
            └── else return false


PERMISSIONS constants (examples):
  MANAGE_TENANTS       → owner only
  MANAGE_BILLING       → owner only
  MANAGE_ADMINS        → owner only
  MANAGE_STUDENTS      → admin+
  MANAGE_TEACHERS      → admin+
  MANAGE_COURSES       → admin+
  MANAGE_PAYMENTS      → admin+
  VIEW_REPORTS         → admin+
  MANAGE_OWN_GROUPS    → teacher+
  TAKE_ATTENDANCE      → teacher+
  CREATE_HOMEWORK      → teacher+
  GRADE_HOMEWORK       → teacher+
  VIEW_OWN_COURSES     → student+
  VIEW_OWN_GRADES      → student+
  SUBMIT_HOMEWORK      → student+
  VIEW_OWN_ATTENDANCE  → student+
```

## Module Structure

Each feature module follows this consistent pattern:

```
modules/<feature>/
├── components/         UI components specific to this feature
│   ├── <Feature>Table.tsx       Desktop table view
│   ├── <Feature>CardList.tsx    Mobile card view
│   ├── <Feature>Form.tsx        Create/edit form (shared)
│   └── <Feature>Detail.tsx      Detail view / drawer
├── hooks/
│   ├── use<Feature>s.ts         List query hook
│   ├── use<Feature>.ts          Single item query hook
│   ├── useCreate<Feature>.ts    Create mutation hook
│   ├── useUpdate<Feature>.ts    Update mutation hook
│   └── useDelete<Feature>.ts    Delete mutation hook
├── mappers/
│   └── <feature>.mapper.ts      DTO ↔ FormValues transformation
├── types/
│   └── <feature>.types.ts       Frontend-only types
└── index.ts                     Public barrel export
```

Query hooks in `hooks/` are thin wrappers over generated hooks that add:
- Tenant-scoped query keys
- Error toast side effects
- Sensible stale/cache time defaults
- TypeScript narrowing of the return type

Mappers in `mappers/` handle the impedance mismatch between:
- Backend DTOs (nested objects, ISO timestamps, snake_case IDs)
- Form values (flat objects, date strings, human-readable)
- Display models (formatted strings, computed booleans like `isOverdue`)
